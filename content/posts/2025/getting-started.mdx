# Next.js 16 블로그 시작하기

Next.js 16의 App Router를 활용하여 개인 기술 블로그를 구축하는 방법을 소개합니다.

## 왜 Next.js 16인가?

Next.js 16은 React 19와 완벽하게 통합되며, 다음과 같은 장점을 제공합니다:

- **서버 컴포넌트**: 기본적으로 서버에서 렌더링되어 성능 향상
- **정적 생성**: `output: 'export'`로 완전한 정적 사이트 생성
- **타입 안정성**: TypeScript 5.x 완벽 지원
- **최적화**: 자동 코드 분할 및 이미지 최적화

<Callout type="info" title="참고">
이 블로그는 GitHub Pages에서 무료로 호스팅됩니다!
</Callout>

## 프로젝트 생성

다음 명령어로 Next.js 프로젝트를 생성합니다:

```bash
npx create-next-app@latest blog --typescript --tailwind --app
cd blog
npm install
```

## 핵심 기능

### 1. MDX 지원

MDX는 마크다운에 JSX를 결합한 형식으로, 블로그 포스트 작성에 최적화되어 있습니다.

```typescript title="lib/mdx.ts"
import { compileMDX } from 'next-mdx-remote/rsc'
import remarkGfm from 'remark-gfm'
import rehypePrettyCode from 'rehype-pretty-code'

export async function getPostBySlug(slug: string) {
  const { content } = await compileMDX({
    source: fileContent,
    options: {
      mdxOptions: {
        remarkPlugins: [remarkGfm],
        rehypePlugins: [[rehypePrettyCode, options]],
      },
    },
  })

  return content
}
```

### 2. 코드 하이라이팅

Shiki를 사용하여 아름다운 코드 블록을 생성합니다:

```javascript title="example.js" {3-5}
function greet(name) {
```

## 배포

GitHub Pages 배포는 GitHub Actions를 통해 자동화됩니다:

```yaml title=".github/workflows/deploy.yml"
name: Deploy to GitHub Pages

on:
  push:
    branches: [main]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
      - run: npm ci
      - run: npm run build
      - uses: actions/upload-pages-artifact@v3
        with:
          path: ./out
```
